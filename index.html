A/B Testing Assignment

The Assignment

For this assignment, I had to run a simple A/B test on two different UI designs and analyze the results. From the results, I will draw conclusions on whether a design choice could hvae reasonably impacted user interaction.

Two Interfaces

Data Collection: Interface A

To start, as a class, we all used Interface A to make an appointment with Adam Ng, MD at Morristown Medical Center on April 23, 2024. Then, we were given 30 minutes to make changes to Interface A that would impact the user's flow in completing the task.

The changes I made to Interface A for Interface B:

  • correcting the order of dates
  • adding a grey background to see which appointment the user is hovering over
  • making the font darker
  • making the buttons for appointments more notciable
  • adding a hover feature to the buttons to show when the user is hovering over a button

Interface B

Data Collection: Interface B

After we finished our changes, as a class, we all used Interface B to make an appointment with Adam Ng, MD at Morristown Medical Center on April 23, 2024.

I then saved all the data from the uses of the two Interfaces to perform statistical analysis on whether or not the changes I made were significant.

Analysis

For the analysis portion of this assignment, we were tasked with creating three hypotheses to test. We had to create hypotheses about misclick rate and time on page, and were given a choice for the third hypothesis we were to test. I chose to hypothesize about and test the time to first click on the page because I made changes to my interface that I hoped would allow the user to find their desired appointment quicker, leading to a quicker time to first click on page B than page A.

Hypothesis 1: Misclick rate

H_0 = There were the same amount of misclicks on page A and page B.

H_1 = There were fewer misclicks on page B than page A.

The reason behind this alternative hypothesis is that I want to see that my changes on Interface B led to fewer misclicks than Interface A.

Predictions

I predicted that we will be able to reject the null hypothesis because I designed Interface B to show different colors for both the appointment slots and the buttons, hopefully showing the user where to correctly click to make an appointment.

Statistical Test and Results

For this hypothesis, I performed a chi squared test to measure the differences in frequencies of misclicks.

  • chi-squared statistic = 31.613
  • p-value = 0.001

Because the p-value is less than 0.05, we can reject the null hypothesis, determining that there is significant difference between the frequencies of misclicks on page A and page B.

Hypothesis 2: Time on page

H_0 = Users spent the same amount of time on each page.

H_1 = Users spent less time on page B than page A.

The reason behind this alternative hypothesis is that I want to see that my changes on Interface B led to users not having to spend as much time searching the page to get to their appointment.

Predictions

I predicted that users will spend less time on Interface B than Interface A because they will have an easier time finding their appointment.

Statistical Test and Results

For this hypothesis, I performed a two tailed t-test to determine whether or not there is statistical difference between the means of the two groups of data.

  • T-score = 4.609
  • P-value = 0.001
  • Degrees of Freedom = 23.289

Since the p-value is less than 0.05, we can reject the null hypothesis and determine that there is statistical difference between the mean of time spent on page A and the mean of time spent on page B.

Hypothesis 3: Time to first click

H_0 = The time to first click on page A and page B were the same.

H_1 = The time to first click on page B was faster than the time to first click on page A.

The reason behind this alternative hypothesis is that I want to see that my changes on Interface B led to users finding their appointment quicker without clicking on elements of the page that were not helpful to making their appointment.

Predictions

I predicted that the null hypothesis will be rejected and we will find statistical difference between the time to first click for Interface A and Interface B because Interface B was created in a way that it made the process of scheduling an appointment more obvious to the user in hopes to reduce the time to the first click.

Statistical Test and Results

For this hypothesis, I performed a two tailed t-test to determine if there is statistical difference in the means of time to first click on each interface.

  • T-score = 5.233
  • P-value = 0.001
  • Degrees of Freedom = 22.035

Since the p-value is less than 0.05, we can reject the null hypothesis that the time to the first click on page A and page B were equal, noting that there is statistical difference between the means of the time to first click for page A and page B.

Summary

Through hypothesis testing, I can determine that Interface B is better than Interface A for fewer misclicks, less time spent on the page, and less time to first click. The changes that I made to Interface A for Interface B led to changes in the way that users interacted with this interface. Some pitfalls of this type of A/B testing is that there were not many responses to the interfaces were we can acutally determine if the changes made were significant. We would need to test on a much larger scale and with people who are not familiar with the interfaces in order to perform real A/B testing. Overall, this was a great experience to be able to learn about the process of A/B testing, how to create hypotheses to test, choosing correct tests for hypotheses, and analyzing results.
